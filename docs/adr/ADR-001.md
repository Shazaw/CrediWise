# ADR-001: Native iOS (SwiftUI) as the MVP client

- **Status:** Accepted
- **Date:** 2026-07-16

## Context

CrediWise needs one polished, demo-ready client for the hackathon (PLAN §1.2
O4) that can render financial dashboards, charts, and interactive shock
simulations with a native feel.

## Decision

Build the MVP client as a **native iOS app in SwiftUI (iOS 16+)**, using
Swift Concurrency and Swift Charts (PLAN §9, §13).

## Alternatives Considered

- **React Native / Flutter:** faster cross-platform reach, but weaker native
  financial-UX feel and no stakeholder preference for it.
- **Web app:** fastest to ship across devices, but the stakeholder explicitly
  prioritized a native iOS demo experience.

## Consequences

- Android and a web lender console are explicitly post-MVP (PLAN §1.5, §6.4).
- iOS-specific tooling (Xcode, TestFlight) is required in CI/CD for the
  frontend workstream.
- The backend API must be platform-agnostic (JSON over HTTPS) so a future
  Android or web client can reuse it without backend changes.
