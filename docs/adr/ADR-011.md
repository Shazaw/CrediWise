# ADR-011: Guided stepper navigation (not a tab bar) for MVP

- **Status:** Accepted
- **Date:** 2026-07-16

## Context

The MVP demo is a single scripted golden path — need → upload → processing →
dashboard → offers (PLAN §1.6) — that must run reliably 10/10 times during
judging (PLAN §1.3).

## Decision

Use a **guided stepper** as the primary post-login flow (PLAN §13.3), not a
tab bar. A home tab bar is deferred to post-MVP.

## Alternatives Considered

- **Tab bar with free navigation:** more typical of a mature app and better
  for repeat, non-linear usage, but it lets a demo run wander off the
  scripted path and increases the surface area for an incomplete-screen
  demo failure.

## Consequences

- Coordinator/routing code is optimized for a linear sequence with typed
  routes (PLAN §13.3), not a hub-and-spoke navigation graph.
- Introducing a tab bar post-MVP is a navigation-layer change, not a data or
  API change — no backend impact.
