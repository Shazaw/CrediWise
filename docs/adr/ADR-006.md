# ADR-006: LLM produces explanations only — deterministic engines decide

- **Status:** Accepted
- **Date:** 2026-07-16

## Context

CrediWise's scores (Data Confidence, Risk Band, Shock Resilience, Safe
Offer) must be defensible to a regulator, reproducible, and fair across
income-volatility segments (PLAN §19.4). An LLM's non-determinism and
opacity are incompatible with that bar.

## Decision

**Deterministic, version-controlled rule engines** (PLAN §15, `engines/`)
compute every numeric score. AI (local Kimi and any explanation adapter) may
only interpret evidence or phrase already-computed reason codes — it can
never invent, alter, or independently produce a score (PLAN §4.5 A5,
CLAUDE.md §4.3).

## Alternatives Considered

- **LLM-scored risk/affordability:** faster to prototype and more "flexible"
  to edge cases, but non-reproducible, hard to audit, and impossible to
  defend as a consistent, fair decision process to a regulator or a
  disputing user.

## Consequences

- Every score change requires a new `model_version` and golden-fixture
  review (PLAN §15.4, §19.2) — never a prompt change.
- The explanation adapter's prompt contract is strict: "Explain, do not
  decide. Do not invent numbers. Use provided reason codes only." (PLAN
  §15.5).
- A deterministic template fallback is required so the LLM being unavailable
  never blocks the dashboard (PLAN §15.5, §20.6).
