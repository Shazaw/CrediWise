# ADR-003: Custom JWT + RBAC authentication (argon2id)

- **Status:** Accepted
- **Date:** 2026-07-16

## Context

CrediWise's core differentiator is a **consent-scoped, role-aware**
authorization model: users, lenders, reviewers, and admins each see
different slices of the same assessment (PLAN §3.2, §18.3). Authorization
must be enforced per-resource (ownership) and per-grant (active consent),
not just per-role.

## Decision

Implement **custom JWT authentication** (short-lived access token + rotating
refresh token, RS256 signing, `argon2id` password hashing) with a reusable
`require(role, ownership, consent)` dependency (PLAN §18.1–§18.3).

## Alternatives Considered

- **Managed auth (Supabase Auth, Auth0):** much faster to integrate, but
  their role/permission models do not natively express "this lender may read
  this user's package only while an unexpired consent grant exists" — the
  core two-way product mechanic (PLAN §2). Bolting consent-scoping on top of
  a managed provider would fight the tool rather than use it.

## Consequences

- Full control over the authorization guard used on every endpoint
  (deny-by-default, PLAN §18.3, §24.1).
- The team owns token revocation, rotation, and rate-limiting logic that a
  managed provider would otherwise supply.
- Refresh tokens are stored server-side (hashed) specifically so they can be
  revoked on logout or suspected breach.
