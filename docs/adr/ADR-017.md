# ADR-017: CrediWise-owned repayment model as shadow research evidence

- **Status:** Accepted
- **Date:** 2026-07-17

## Context

CrediWise needs to demonstrate an internally designed and trained repayment
model while preserving the product's deterministic safety guarantees. No
verified public borrower-level Indonesian repayment dataset is available.
Available public datasets differ materially in geography, target, observation
process, and features. Pooling them or presenting foreign benchmark performance
as Indonesian predictive validity would be misleading.

The Berka/PKDD'99 Financial dataset is the closest public source to CrediWise's
transaction-derived vision because it links dated account transactions to loan
outcomes. Its standard completed-loan A-vs-B task has only 234 eligible loans
and 31 adverse outcomes. The CTU repository does not state an explicit
redistribution license.

## Decision

CrediWise owns and versions `crediwise-cashflow-risk:v1-research`, including its
feature schema, acquisition and training code, preprocessing, calibration,
artifact, inference runtime, explanations, tests, and model card. Source data
remains attributed and is not claimed as proprietary CrediWise data.

The model:

- trains only on completed Berka A/B loans;
- uses only transactions strictly before each loan start;
- excludes demographics, names, identifiers, text descriptions, Data
  Confidence, deterministic scores, and post-index evidence;
- exports a canonical, non-executable, SHA-256-validated JSON linear artifact;
- persists an exact feature vector/hash and model/artifact lineage per result;
- runs as `SHADOW_RESEARCH` after deterministic cash-flow analysis;
- is always LOW confidence while based only on the historical foreign
  benchmark;
- degrades to `INELIGIBLE` or `UNAVAILABLE` without blocking the assessment;
- cannot alter any authoritative score, safe borrowing result, offer result,
  pricing, approval, or raw evidence.

The UCI Taiwan and South German datasets are acquired under CC BY 4.0 for
separate benchmark research. They are not pooled with Berka because their
targets and feature semantics are incompatible.

## Alternatives Considered

- **Pool public datasets:** rejected because the targets, decision times,
  populations, and feature meanings are not equivalent.
- **Use PaySim or fraud datasets as repayment labels:** rejected as a category
  error; fraud is not inability to repay.
- **Replace the deterministic Risk Engine:** rejected because public foreign
  data cannot support Indonesian calibration or consumer-safety claims.
- **Load pickle/joblib artifacts in production:** rejected because those formats
  can execute code and couple inference to training-library versions.
- **Hide the model completely:** rejected because a carefully labelled visible
  research card demonstrates the architecture without implying authority.

## Consequences

- The app can accurately state that the CrediWise team designed and trained its
  own model, while attributing public source data.
- A new append-only prediction table and additive model-registry metadata are
  required.
- The experimental probability remains supplementary and visibly distinct from
  the deterministic Indicative Risk Band.
- Promotion requires representative, prospectively governed Indonesian
  repayment outcomes, fixed outcome horizons, out-of-time calibration,
  subgroup analysis, confidence intervals, privacy/fairness review, and a new
  accepted ADR/model version.
