# ADR-002: FastAPI (Python) backend

- **Status:** Accepted
- **Date:** 2026-07-16

## Context

The backend must parse OCR/PDF documents, run deterministic financial
engines, and optionally call a local multimodal AI model for document
anomaly assistance, while remaining strictly typed and testable
(PLAN §9, §10).

## Decision

Build the backend in **Python 3.12 with FastAPI**, Pydantic for request/
response validation, and async-capable routing.

## Alternatives Considered

- **Node.js / NestJS:** strong HTTP ergonomics, but a materially weaker
  document/OCR and data-science ecosystem.
- **Go:** excellent performance, but a poor fit for OCR, PDF forensics, and
  numerical/finance libraries; slower iteration for scoring-engine work.

## Consequences

- The team's existing Python/ML familiarity is directly leveraged.
- Pydantic schemas double as the OpenAPI contract source (PLAN §12.4).
- Deterministic engines (PLAN §15) are implemented as pure Python functions
  with no framework coupling, keeping them swappable for trained models
  later (PLAN §4.4).
