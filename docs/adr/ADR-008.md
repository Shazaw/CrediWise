# ADR-008: Mock bank/lender adapters behind real interfaces

- **Status:** Accepted
- **Date:** 2026-07-16

## Context

CrediWise is not licensed or credentialed to access real bank Open Finance
APIs or real lender systems during the hackathon, but the product's
architecture (and the post-MVP roadmap, PLAN §16.2) depends on those
integrations existing eventually.

## Decision

Implement `FinanceProviderPort` and `LenderOfferPort` interfaces now, backed
in MVP by a `MockOpenFinanceProvider` and a seeded lender catalog, so the UI
and pipeline are already **Open-Finance-shaped** (PLAN §16.1, ADR-008 origin
in §23).

## Alternatives Considered

- **Hard-code mock data directly in services/UI:** faster to write, but
  would require a rewrite (not a swap) when real integrations arrive,
  defeating the "production-shaped, hackathon-scoped" goal (PLAN §1.2 O5).

## Consequences

- Every simulated offer/account is explicitly labelled `SIMULATED` /
  `SIMULATED_REGULATED_PROVIDER` — never implying a real endorsement
  (PLAN §11.3 `lender_offers.offer_source`, FR-11 AC5).
- Swapping a mock adapter for a real one post-pilot is additive: implement
  the port, wire config, no caller changes.
- No real financial-institution or lender production credentials exist in
  any environment (PLAN §4.3 C6, CLAUDE.md §15.1).
